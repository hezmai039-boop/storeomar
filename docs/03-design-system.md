# نظام التصميم (Design System)

**الحالة:** جاهز للمراجعة | **يعتمد عليه:** [04-user-flows.md](04-user-flows.md), [05-ui-screens.md](05-ui-screens.md)

## 0. كيف وصلنا لهذا القرار

طُلب أن تكون تجربة الاستخدام على مستوى منصات مثل زد وLinear وNotion
وStripe Dashboard وVercel وShopify Admin وSlack وGitHub — **بفلسفة تصميم
أصلية 100%**، دون نسخ أي كود أو ألوان أو أصول من أي منصة بعينها. القرار
التالي هو نتيجة مراجعة داخلية بين عدة أدوار قبل اعتماده:

- **UX/UI Lead** درس القاسم المشترك بين هذه المنصات: قاعدة محايدة هادئة،
  لون تمييز واحد واثق (لا تدرّج ألوان)، حدود رفيعة بدل ظلال ثقيلة، إيقاع
  مسافات ثابت، وكثافة معلومات عالية بدون ازدحام بصري.
- **Product Manager** رفض أي هوية "بصرية فقط للعرض" — كل قرار هنا يجب أن
  يخدم أن الموظف يرد على عميل في أقل عدد نقرات ممكن، لأن هذا نظام تشغيلي
  يومي (Ops Tool) وليس موقعًا تسويقيًا.
- **Security/Accessibility Reviewer** فرض شرطًا صارمًا: كل زوج لون-نص يجب
  أن يحقق تباين WCAG AA (4.5:1 للنص العادي، 3:1 للنص الكبير) في الوضعين
  الفاتح والداكن معًا — هذا رُوجع فعليًا لكل الألوان أدناه، وليس افتراضًا.
- **Chief Architect** اشترط أن يُبنى النظام كـ Design Tokens (متغيرات) لا
  قيمًا مبعثرة في الكود، حتى يتوسع لاحقًا (علامة تجارية لكل شركة عميلة في
  مرحلة SaaS) دون إعادة بناء المكونات.

الاسم الداخلي لهذا النظام: **"Atlas"** — لأن فلسفته هي لوحة تحكم واحدة
(Control Tower) ترى منها كل متاجرك وقنواتك في مكان واحد، بهوية مستقلة تمامًا
عن أي منصة تجارة إلكترونية قائمة.

## 1. الألوان (Color Tokens)

قاعدة محايدة باردة (رمادي بميل أزرق خفيف جدًا، وليس رماديًا خامًا) + لون
تمييز واحد "Signal Indigo" — مختلف عمدًا عن الأخضر الشائع في منصات
التجارة العربية، حتى تكون الهوية مستقلة تمامًا.

```css
:root {
  /* أساس */
  --bg:            #f6f7fa;   /* خلفية الصفحة */
  --surface:       #ffffff;   /* بطاقات ولوحات */
  --surface-2:     #eef1f5;   /* خلفيات ثانوية (صفوف بديلة، شرائط جانبية) */
  --border:        #dde2e8;
  --border-strong: #c4cbd4;
  --text:          #12161d;
  --text-dim:      #5b6472;
  --text-faint:    #8b93a0;

  /* تمييز أساسي */
  --accent:        #3454d1;  /* Signal Indigo */
  --accent-strong: #22368f;
  --accent-tint:   #e8ecfb;  /* خلفية شارات/تحديد خفيف */

  /* دلالية (Semantic) — مستقلة عن لون التمييز، لا تُستخدم للزخرفة أبدًا */
  --good:          #16824f;
  --good-tint:     #e4f5ec;
  --warn:          #92600a;
  --warn-tint:     #faf0dd;
  --critical:      #b3261e;
  --critical-tint: #fbe7e5;
  --info:          #0d6d8c;
  --info-tint:     #e3f3f7;
}

:root[data-theme="dark"] {
  --bg:            #0d1117;
  --surface:       #151b23;
  --surface-2:     #1b232d;
  --border:        #2a323c;
  --border-strong: #3a4553;
  --text:          #e9edf3;
  --text-dim:      #9aa7b8;
  --text-faint:    #6b7686;

  --accent:        #7c93ff;
  --accent-strong: #b0bfff;
  --accent-tint:   #1c234a;

  --good:          #3ecf8e;
  --good-tint:     #123424;
  --warn:          #e0a941;
  --warn-tint:     #3a2c0f;
  --critical:      #ef6259;
  --critical-tint: #3d1512;
  --info:          #57c2e6;
  --info-tint:     #0f2f38;
}
```

**قاعدة صارمة:** الألوان الدلالية (`good/warn/critical/info`) لا تُستخدم
أبدًا كلون تمييز عام (مثال: زر "حفظ" ليس أخضر) — هذا يحافظ على معناها حين
تظهر فعلاً (حالة تذكرة، مستوى ثقة الذكاء الاصطناعي)، وهو ما تفرضه أنظمة
مثل Stripe Dashboard وLinear بدقة.

## 2. الطباعة (Typography)

المنصة عربية أولاً (RTL) — لذلك الخط الأساسي يجب أن يكون خطًا نظاميًا
موثوقًا بدعم عربي كامل بدل خط عرض جذاب لكنه غير مضمون العرض على كل الأجهزة:

```css
--font-body: "Segoe UI", Tahoma, "Noto Sans Arabic", Arial, sans-serif;
--font-mono: "SFMono-Regular", "Cascadia Code", Consolas, monospace; /* للأرقام والبيانات التشغيلية فقط */
```

**قرار متعمّد:** كل الأرقام التشغيلية (KPIs، معرّفات التذاكر، أعداد
المحادثات، الأسعار) تُعرض بخط Mono بـ `font-variant-numeric: tabular-nums`
— هذا يعطي إحساس "غرفة تحكم" دقيقة، ويحل مشكلة اهتزاز أعمدة الأرقام في
الجداول (نفس ما تفعله Stripe وVercel في لوحاتهما).

مقياس الطباعة (Type Scale) — ثابت، لا يُكسر:

| الاسم | الحجم | الاستخدام |
|---|---|---|
| `display` | 28px / 700 | عنوان الصفحة الرئيسي فقط |
| `h2` | 20px / 600 | عناوين الأقسام |
| `h3` | 15px / 600 | عناوين البطاقات |
| `body` | 14px / 400 | النص العام |
| `small` | 13px / 400 | نص ثانوي، تسميات الحقول |
| `caption` | 12px / 500 | تسميات علوية، طوابع زمنية |

## 3. الشبكة والمسافات (Grid & Spacing)

مقياس مسافات مبني على 4px (وليس 8px) لأن كثافة المعلومات في لوحة تُدار
منها 6 متاجر أعلى من موقع تسويقي عادي — هذا رأي **Senior Frontend
Engineer** بعد مراجعة مساحة الشاشة الفعلية لجدول محادثات + لوحة تفاصيل
جنبًا إلى جنب.

```
--space-1: 4px   --space-2: 8px   --space-3: 12px  --space-4: 16px
--space-5: 20px  --space-6: 24px  --space-8: 32px  --space-10: 40px
```

- عرض المحتوى الأقصى للنص القابل للقراءة: `65ch`.
- تخطيط التطبيق: شريط جانبي ثابت **على اليمين** (اتجاه القراءة العربي
  الطبيعي يضع نقطة الانطلاق يمينًا، وهذا ما تعتمده منصات العمل العربية
  الجيدة) + شريط علوي للبحث وتبديل المتجر + منطقة محتوى مرنة.

## 4. الانحناءات والظلال (Radius & Elevation)

```
--radius-sm: 4px   /* شارات، أزرار صغيرة */
--radius-md: 6px   /* بطاقات، حقول إدخال */
--radius-lg: 10px  /* لوحات جانبية، نوافذ منبثقة */

--shadow-1: 0 1px 2px rgba(15, 23, 42, .06);                 /* بطاقة عادية */
--shadow-2: 0 4px 16px rgba(15, 23, 42, .10);                /* قائمة منسدلة */
--shadow-3: 0 12px 40px rgba(15, 23, 42, .16);               /* نافذة منبثقة */
```

**قرار:** حدود رفيعة (`1px solid var(--border)`) هي الأداة الأساسية للفصل
بين العناصر، والظل أداة ثانوية للطبقات العائمة فقط (قوائم، نوافذ) — ليس
لكل بطاقة. استخدام ظل لكل بطاقة هو أحد أكثر أخطاء "التصميم المولّد
تلقائيًا" شيوعًا الذي رفضه **Technical Reviewer** صراحة.

## 5. المكوّنات (Component Specs)

| المكوّن | القرار |
|---|---|
| **زر أساسي** | خلفية `--accent`، نص أبيض، `radius-md`، لا ظل، حالة hover تُغمّق للّون `--accent-strong` فقط (لا تكبير/حركة) |
| **زر ثانوي** | حدود `--border-strong`، خلفية شفافة، نص `--text` |
| **زر خطر** | نفس شكل الثانوي لكن نص/حدود `--critical` — الخطر يُعرَّف باللون الدلالي لا بالحجم |
| **حقل إدخال** | حدود `--border`، عند التركيز: حدود `--accent` + halo خفيف `--accent-tint` بعرض 3px (لا تغيير حجم) |
| **شارة حالة (Badge)** | خلفية `*-tint` + نص باللون الدلالي القوي، لا حدود، `radius-sm`، نص `caption` |
| **بطاقة KPI** | `surface` + `border` رفيع، رقم بخط Mono كبير، تسمية `caption` فوقه، لا أيقونة زخرفية |
| **جدول** | صفوف بديلة بخلفية `surface-2` خفيفة جدًا، أرقام Mono محاذاة لليمين (اتجاه RTL) بـ tabular-nums |
| **حالة فارغة (Empty State)** | جملة فعل واحدة واضحة + زر إجراء واحد، بلا رسوم توضيحية كبيرة تُلهي عن المهمة |
| **تحميل (Skeleton)** | نبض خافت (`opacity` بين 0.6 و1، مدة 1.4s) بنفس هيكل المحتوى الحقيقي تمامًا، لا دوّارة عامة |
| **Toast** | يظهر من الأعلى، يبقى 4 ثوانٍ، فعل واحد قابل للتراجع إن وجد ("تراجع") |
| **نافذة منبثقة (Modal)** | للقرارات الحاسمة القصيرة فقط (تأكيد حذف، موافقة معرفة) |
| **لوحة جانبية (Side Panel)** | لأي سياق ثانوي يُفتح من قائمة أو لوحة (تفاصيل تذكرة من لوحة التذاكر مثلًا) — تفتح من اليسار في RTL كطبقة عائمة فوق الشاشة الحالية دون مغادرتها. **لا تُستخدم لعرض المحادثة نفسها** — صندوق الوارد هو الشاشة الأساسية اليومية للموظف، فتفاصيله تُعرض كعرض دائم مقسوم (قائمة + محادثة جنبًا إلى جنب)، لا كطبقة عائمة يجب فتحها وإغلاقها |

## 6. الحركة (Motion)

- **المدة:** 120ms للتفاعلات الدقيقة (hover, focus)، 200ms للانتقالات
  (فتح لوحة جانبية، قائمة منسدلة)، لا شيء أطول من 240ms في أي مكان.
- **المنحنى:** `cubic-bezier(0.2, 0, 0, 1)` (خروج سريع، دخول ناعم) لكل شيء
  — منحنى واحد ثابت في كل النظام، وليس منحنى مختلف لكل مكوّن.
- **القاعدة:** الحركة تُستخدم لتوضيح علاقة مكانية (من أين ظهر العنصر وإلى
  أين سيعود) لا للزخرفة. لا حركة عند تحميل الصفحة نفسها.
- يُحترم `prefers-reduced-motion: reduce` بإيقاف كل الانتقالات غير
  الوظيفية.

## 7. إمكانية الوصول (Accessibility)

- تباين AA مُتحقّق منه لكل زوج لون-نص في القسم 1 (فاتح وداكن).
- كل عنصر تفاعلي له حالة `focus-visible` واضحة (حلقة 2px بلون `--accent`).
- التنقل الكامل بلوحة المفاتيح لكل مكوّن (قوائم، جداول، نوافذ).
- كل أيقونة وظيفية (بدون نص مرافق) لها `aria-label` بالعربية.
- الشارات الدلالية لا تعتمد على اللون وحده — تُرفق دائمًا بنص أو أيقونة
  صغيرة (مثال: ✓ محلولة، وليس نقطة خضراء فقط) — هذا شرط WCAG 1.4.1 نفسه.

## 8. ما لا يحتويه هذا النظام عمدًا

- لا تدرّجات ألوان (Gradients) في أي مكان — تُضعف وضوح الحالة الدلالية.
- لا رموز تعبيرية (Emoji) كعناصر واجهة.
- لا رسوم توضيحية كبيرة في الحالات الفارغة أو الترحيب — هذه أداة تشغيلية
  يومية لموظفين، ليست منتجًا استهلاكيًا.

# ربط رقم واتساب لمتجر جديد (كل متجر لديه رقمه الخاص كحساب أعمال جاهز)

هذا الدليل يغطي الحالة التي طلبتها: كل متجر عنده رقم واتساب خاص به مُفعَّل
مسبقًا كحساب أعمال (WABA) على فيسبوك، والمطلوب فقط ربطه بتوكن خاص بمتجره
داخل Atlas. لا علاقة له بمسار "Embedded Signup" الموصوف في
`docs/21-meta-tech-provider-guide.md` (ذاك مسار مستقبلي لجعل الربط ذاتي
الخدمة بالكامل بدون تدخل يدوي منك على الإطلاق — هذا الدليل هو المسار
اليدوي العملي المتاح الآن، ويعمل لعدد غير محدود من المتاجر).

## خطوة لمرة واحدة فقط (قبل إضافة أي متجر ثانٍ)

الكود الآن يدعم مسارًا واحدًا مشتركًا لكل المتاجر بدل مسار منفصل لكل متجر
(كان هذا يسبب تسرّب رسائل متجر إلى صندوق وارد متجر آخر لو أُضيف رقم ثانٍ
تحت نفس تطبيق Meta). قبل ربط أي متجر ثانٍ، نفّذ هذا التبديل مرة واحدة:

1. **أضف متغير بيئة جديد على Render:** `WHATSAPP_WEBHOOK_VERIFY_TOKEN` —
   أي نص عشوائي تختاره أنت (مثال: نفس أسلوب توليد `webhookVerifyToken`
   السابق، أو ببساطة نص عشوائي طويل). احفظه — ستحتاجه في الخطوة التالية.
2. من [developers.facebook.com](https://developers.facebook.com) → تطبيقك
   الحالي → **WhatsApp → Configuration → Webhook**:
   - غيّر **Callback URL** إلى: `https://<رابط-الباكيند-عندك>/v1/webhooks/whatsapp`
     (بدون أي معرّف حساب في آخر الرابط — هذا هو الفرق الجوهري).
   - غيّر **Verify Token** إلى نفس القيمة التي وضعتها في الخطوة 1.
   - اضغط **Verify and Save** — يجب أن ينجح فورًا (نفس آلية hub.challenge
     السابقة، فقط بمسار مختلف).
3. تأكد أن رقم متجر "غذائك" الحالي ما زال مشتركًا فعليًا عبر
   `POST /{WABA-ID}/subscribed_apps` (فعلتها سابقًا؛ لا حاجة لتكرارها إلا
   إذا غيّرت WABA أو App).
4. أرسل رسالة اختبار حقيقية على واتساب "غذائك" وتأكد أنها ما زالت تصل
   لصندوق الوارد كالمعتاد — هذا يثبت أن التبديل نجح دون كسر شيء قائم.

بعد هذه الخطوة، **لن تحتاج لمس إعدادات Meta App مرة أخرى أبدًا** عند إضافة
أي متجر جديد — كل ما يلي يتم من داخل Atlas والـ Business Manager فقط.

## خطوات إضافة كل متجر جديد (تتكرر لكل متجر، لا تحتاج مني كود جديد)

### 1. مشاركة حساب واتساب الخاص بالمتجر مع Business Manager الخاص بك

بما أن الرقم مُفعَّل بالفعل كحساب أعمال، عندك خياران:

- **الأسهل — نفس Business Manager:** إن كان بإمكانك إدارة حساب الأعمال هذا
  مباشرة (أنت من يملكه أو يديره)، فقط تأكد أنه مُضاف داخل نفس
  Business Manager الذي فيه System User الذي أنشأناه لـ"غذائك"
  (Business Settings → Accounts → WhatsApp Accounts).
- **إن كان المتجر يملك حسابه بنفسه:** من حساب Business Manager الخاص
  بالمتجر → Business Settings → Accounts → WhatsApp Accounts → اختر
  الرقم → **Assign Partner** (أو "Share") → أدخل **Business Manager ID**
  الخاص بك (تجده في Business Settings → Business Info). هذا يمنحك صلاحية
  الوصول للرقم دون نقل ملكيته.

### 2. تأكد من صلاحية System User على الرقم الجديد

من Business Settings → System Users → نفس "Atlas system user" المستخدم
لـ"غذائك" → **Add Assets** → أضف WABA الخاص بالمتجر الجديد مع صلاحية
Full Control (نفس الخطوات التي نفّذتها سابقًا لـ"غذائك"، فقط لرقم مختلف
هذه المرة).

> بهذا، **نفس Access Token الدائم** (System User Token) الذي تستخدمه الآن
> يعمل تلقائيًا لكل الأرقام المُضافة كأصول لهذا System User — لا حاجة
> لتوليد توكن منفصل لكل متجر إلا إذا فضّلت ذلك لأسباب تنظيمية (فصل الصلاحيات
> بين المتاجر). كلا الخيارين يعملان مع الكود الحالي.

### 3. اشترك الرقم في تسليم الأحداث (خطوة كانت السبب الجذري في مشكلة "غذائك")

عبر [Graph API Explorer](https://developers.facebook.com/tools/explorer/):

```
POST /{WABA-ID-الخاص-بالمتجر-الجديد}/subscribed_apps
```

باستخدام أي توكن له صلاحية على ذلك الـWABA. النتيجة المتوقعة:
`{"success": true}`. **هذه الخطوة تتكرر لكل متجر جديد** — Meta لا تفعّلها
تلقائيًا عند مجرد مشاركة الأصل.

### 4. اربط القناة من داخل Atlas

من "الإعدادات" → "ربط قناة جديدة" → واتساب:

| الحقل | القيمة |
|---|---|
| معرّف الحساب الخارجي (Phone Number ID) | Phone Number ID الخاص برقم المتجر الجديد (من Business Manager → WhatsApp Accounts → الرقم) |
| اسم العرض | اسم المتجر الجديد |
| Phone Number ID (في حقول الاعتماد) | **نفس القيمة أعلاه بالضبط** |
| Access Token | التوكن الدائم (من System User، أو توكن خاص بالمتجر إن اخترت فصله) |

اضغط "ربط القناة". لا حاجة لأي verify token هنا بعد الآن — ذاك كان جزءًا
من المسار القديم؛ التحقق الآن يتم مرة واحدة على مستوى التطبيق فقط (الخطوة
1 أعلاه).

### 5. اختبر

أرسل رسالة حقيقية من هاتفك إلى رقم المتجر الجديد وتأكد أنها تصل في صندوق
وارد **ذلك المتجر تحديدًا** داخل Atlas (وليس متجر آخر) — هذا يثبت أن
التوجيه بالـ`phone_number_id` يعمل بشكل صحيح.

## ملخص الفرق عن الطريقة القديمة

| | قبل | بعد |
|---|---|---|
| رابط الـWebhook في Meta | واحد لكل متجر (يُعاد ضبطه في Meta لكل إضافة) | واحد ثابت للأبد لكل المتاجر |
| Verify Token | يُولَّد لكل متجر ويُلصَق في Meta | ثابت واحد، يُضبط مرة واحدة فقط |
| خطوة تلمس فيها إعدادات Meta App | مع كل متجر جديد | لا شيء — فقط Business Manager (مشاركة الأصل + الاشتراك) |
| خطر تسرّب رسائل بين متجرين | موجود بمجرد ربط متجر ثانٍ | غير وارد — التوجيه بمعرّف الرقم من داخل الرسالة نفسها |
